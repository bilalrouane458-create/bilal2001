Réalisation n°1 – Isolation et sécurisation du réseau boutique

VLAN 50 & Sécurité des douchettes Wi-Fi

Contexte : Stadium – Boutique officielle du stade | Mise en place d’un réseau isolé pour les caisses et scanners
Télécharger le dossier complet (PDF)

Conditions de réalisation

Ressources fournies : Matériel et logiciel fournis par l’école IRIS, cahier des charges Stadium.

Résultats attendus : Réseau de la boutique entièrement isolé via VLAN 50 avec ACLs strictes, garantissant la sécurité des données commerciales et le bon fonctionnement des douchettes sur le Wi-Fi métier "Shop-Wifi".

Ressources matérielles & logicielles

Matériel

Routeur Cisco 2821, Switch Catalyst 2960, Borne WiFi AIR-CAP3502I, Serveur DELL PowerEdge R740

Logiciels

VMWare vSphere ESXi 8.1, Windows Server 2022, Windows 11, Debian 12, OCS/GLPI, Zimbra, Snort, PuTTY, Wireshark

Description de la réalisation

Un VLAN 50 « Boutique » a été mis en place avec des ACLs strictes pour l'isoler. Un groupe Active Directory "Vendeurs-Boutique" a été créé. Une GPO a été déployée pour bloquer les ports USB sur les postes de caisse (prévention de fuite de données) et pour mapper le lecteur B : vers la base de données des articles. Un Wi-Fi métier "Shop-Wifi" a été déployé pour les douchettes.

Étapes de la mise en place

  1. Isolation réseau : Configuration du VLAN 50 et d'ACLs pour isoler complètement le réseau de la boutique.
  2. Configuration AD : Création du groupe de sécurité "Vendeurs-Boutique" pour gérer les accès.
  3. Déploiement de la GPO de sécurité : Application d'une GPO qui mappe le lecteur B: et désactive les ports USB sur les terminaux de caisse.
  4. Déploiement du Wi-Fi : Mise en service du SSID "Shop-Wifi" pour connecter les douchettes de scan au VLAN 50.

Compétences mobilisées

Wi-Fi IoT (douchettes) GPO prévention fuite de données Blocage USB Segmentation VLAN ACLs Cisco Active Directory

Réalisation n°2 – Supervision, inventaire GLPI et alerting Nagios

GLPI / OCS + Nagios & Zimbra

Contexte : Stadium – Gestion des douchettes, supervision Wi-Fi et communication logistique
Télécharger le dossier complet (PDF)

Conditions de réalisation

Ressources fournies : Infrastructure IRIS, cahier des charges Stadium, accès aux équipements réseau et serveurs.

Résultats attendus : Chaque douchette est inventoriée automatiquement dans GLPI via l'agent OCS, et toute panne de la borne Wi-Fi déclenche une alerte Nagios vers la DSI pour intervention rapide.

Ressources matérielles & logicielles

Matériel

Routeur Cisco 2821, Switch Catalyst 2960, Borne WiFi AIR-CAP3502I, Serveur DELL PowerEdge R740

Logiciels

VMWare vSphere ESXi 8.1, Windows Server 2022, Debian 12, OCS/GLPI, Zimbra, Nagios, Snort, PuTTY, Wireshark

Description de la réalisation

L'inventaire des douchettes est géré dans GLPI grâce aux remontées de l'agent OCS (identification par adresse MAC). Les vendeurs disposent de comptes Zimbra pour la communication logistique. La borne "Shop-Wifi" est supervisée par Nagios, qui alerte la DSI en cas de panne. Une base de connaissances dans GLPI a été créée pour documenter les pannes courantes des scanners ("Panne scan").

Étapes de la mise en place

  1. Inventaire OCS/GLPI : L'agent OCS remonte l'inventaire des douchettes, permettant de les suivre individuellement dans GLPI.
  2. Communication Zimbra : Création de comptes de messagerie pour les vendeurs afin de coordonner les réapprovisionnements.
  3. Supervision Nagios : Surveillance de l'état de la borne Wi-Fi de la boutique avec alertes en cas de panne.
  4. Base de connaissances GLPI : Création d'articles pour aider à la résolution rapide des problèmes fréquents des douchettes.

Compétences mobilisées

OCS/GLPI Inventaire automatisé Base de connaissances ITIL Nagios supervision Alerting DSI Zimbra collaboration Surveillance Wi-Fi
